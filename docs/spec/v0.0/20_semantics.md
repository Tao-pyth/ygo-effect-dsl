# ygoEffectDSL Spec v0.0 (Draft) — Semantics

この文書は **DSLの意味論（評価順序・処理モデル）** を定義します。

## 1. 評価順序（基本）
v0.0 の基本順序は次の通り：
1. trigger（発動契機）
2. restriction（制限：1ターン1度、発動条件など）
3. condition（追加条件：盤面条件など）
4. cost（コスト：支払い）
5. action（解決：効果適用）

## 2. State Transition（状態遷移）モデル
- State S から、適用可能な効果集合 E(S) を列挙する
- e ∈ E(S) を適用し、次状態 S' = apply(S, e) を生成する
- 探索は深さ制限付き（bounded search）を基本とする

## 3. 冪等性と決定性
- 同じ入力（状態・効果・選択）からは同じ出力状態が生成されること（決定性）
- ただし対象選択が分岐する場合は「選択」を明示的に入力として扱う

## 4. v0.0の簡略化（重要）
- 相手のレスポンス無し（自ターン展開のみ）
- チェーン/スタックの厳密再現は v0.0では対象外
