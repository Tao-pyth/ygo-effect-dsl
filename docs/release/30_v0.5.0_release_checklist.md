# Package 0.5.0 Release Checklist

Status: mandatory cumulative source milestone gate

## Reviewed source

- package version、README、roadmap、release stage index、CHANGELOG、governance policyが`0.5.0`で一致する。
- `0.4.0`はtagなしのsuperseded実装stageとし、その変更を`0.5.0`のCHANGELOGへ累積収録する。
- v0.4.0/v0.5.0 milestoneのopen Issueが0件で、未完了作業は理由、label、milestoneを揃えて後続releaseへ移動している。
- Experiment、Route DSL、Replay、Search、analytics contractのversionをpackage番号へ機械的に追従させていない。
- statistical pruning、localized card source、未検証timing、node-level pool/cache、一般配布を完了済みと主張しない。

## Verification

1. `python -m pytest -q`を完走する。
2. changed Python fileのRuffと`git diff --check`を通す。
3. `python -m build`で`0.5.0` wheelとsdistを作る。
4. `python -m ygo_effect_dsl.release_audit dist/ygo_effect_dsl-0.5.0-py3-none-any.whl dist/ygo_effect_dsl-0.5.0.tar.gz`でpathとthird-party payload境界を検査する。
5. clean virtual environmentへwheelをinstallし、`--version`、`experiment-search --help`、`job-inspect --help`を実行する。
6. `python -m ygo_effect_dsl.release_governance --check`でGitHub Issue label/milestone driftが0件であることを確認する。
7. GitHub ActionsのTestsとBuild Windows exeを同一commitで成功させる。

## Publication boundary

- review済みPRをmergeし、`main`を同期してからannotated `v0.5.0` tagを作る。
- tag workflowのTestsとBuild Windows exeが成功したことを[#255](https://github.com/Tao-pyth/ygo-effect-dsl/issues/255)へ記録する。
- tag成功後にv0.4.0とv0.5.0 milestoneをcloseする。
- GitHub Release、PyPI、installer配布、third-party asset公開は行わない。source tagは#91/#134の承認を代替しない。
- tagは移動・再利用しない。失敗時は原因を修正し、新しいpatch versionで回復する。
